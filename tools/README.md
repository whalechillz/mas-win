## 🏷️ 테스트 도구 분류

### **영구 테스트 도구** (`tools/test/`)
- **용도**: 정기적으로 사용하는 테스트 도구
- **예시**: SMS 테스트, API 테스트, 성능 테스트
- **관리**: Git에 커밋, 문서화 유지

### **디버깅 도구** (`tools/debug/`)
- **용도**: 문제 해결 시 사용하는 도구
- **예시**: 로그 뷰어, 에러 추적기, 성능 프로파일러
- **관리**: 필요 시에만 사용, 문제 해결 후 정리

### **임시 테스트 파일** (`tools/temp/`)
- **용도**: 일회성 테스트나 실험용 파일
- **예시**: 임시 스크립트, 실험용 코드
- **관리**: `.gitignore`에 포함, 정기적 삭제

## 📋 테스트 도구 라이프사이클

### **1. 생성 단계**
```bash
# 새로운 테스트 도구 생성
mkdir tools/test/new-test-tool
# 또는
mkdir tools/debug/new-debug-tool
```

### **2. 사용 단계**
- 테스트 도구 사용
- 결과 기록
- 문제 발견 시 디버깅 도구 활용

### **3. 정리 단계**
```bash
# 임시 파일 정리
rm -rf tools/temp/*

# 사용하지 않는 디버깅 도구 정리
rm -rf tools/debug/unused-tool
```
