# 고객 이미지 방문일자 수정 위치 개선 제안

## 현재 위치의 문제점

### 현재 구현 위치
- **이미지 확대 모달 하단**에 방문일자 수정 UI가 위치
- 이미지를 클릭하여 확대 모달을 열어야만 수정 가능

### 문제점
1. **접근성 낮음**: 이미지를 확대해야만 수정 가능 (2단계 클릭)
2. **메타데이터 오버레이와 겹침**: 이미지 하단에 있어서 메타데이터 오버레이와 겹칠 수 있음
3. **일괄 수정 불가**: 한 번에 하나의 이미지만 수정 가능
4. **시각적 혼란**: 이미지 하단에 UI가 있어서 이미지 감상에 방해될 수 있음

## 개선 방안

### 옵션 1: 썸네일 그리드에서 직접 수정 (권장) ⭐⭐⭐

**위치**: 썸네일 하단 또는 호버 시 표시

**장점**:
- ✅ 빠른 접근 (확대 모달 열 필요 없음)
- ✅ 여러 이미지를 빠르게 수정 가능
- ✅ 목록에서 바로 확인 및 수정
- ✅ 기존 "목록 제거" 버튼과 일관된 UX

**구현 방법**:
1. 썸네일 하단에 작은 날짜 표시
2. 호버 시 "수정" 버튼 표시
3. 클릭 시 인라인 날짜 선택 또는 작은 팝업

**UI 예시**:
```
┌─────────────┐
│   [이미지]   │
│             │
│             │
├─────────────┤
│ 2026-01-26  │ ← 날짜 표시
│ [수정]      │ ← 호버 시 표시
└─────────────┘
```

### 옵션 2: 썸네일 우클릭 메뉴

**위치**: 썸네일 우클릭 시 컨텍스트 메뉴

**장점**:
- ✅ UI가 깔끔함 (항상 표시되지 않음)
- ✅ 여러 액션을 한 곳에 모음

**단점**:
- ❌ 모바일에서 사용 불가
- ❌ 우클릭이 직관적이지 않을 수 있음

### 옵션 3: 별도의 일괄 편집 모달

**위치**: 썸네일 그리드 상단에 "일괄 편집" 버튼

**장점**:
- ✅ 여러 이미지를 한 번에 수정 가능
- ✅ 날짜별 그룹 이동 등 고급 기능 가능

**단점**:
- ❌ 추가 모달 필요
- ❌ 단일 이미지 수정에는 과함

### 옵션 4: 이미지 확대 모달 상단으로 이동

**위치**: 이미지 확대 모달 상단 (닫기 버튼 옆)

**장점**:
- ✅ 메타데이터 오버레이와 겹치지 않음
- ✅ 이미지 감상에 방해되지 않음

**단점**:
- ❌ 여전히 확대 모달을 열어야 함 (접근성 낮음)

## 권장 방안: 옵션 1 (썸네일 그리드에서 직접 수정)

### 구현 계획

#### Phase 1: 썸네일 하단에 날짜 표시
- 각 썸네일 하단에 현재 방문일자 표시
- 작은 폰트, 회색 텍스트

#### Phase 2: 호버 시 수정 버튼 추가
- 기존 "목록 제거" 버튼과 함께 표시
- 클릭 시 인라인 날짜 선택 또는 작은 팝업

#### Phase 3: 인라인 날짜 수정 UI
- 썸네일 하단에 날짜 입력 필드 표시
- 저장/취소 버튼

### 코드 수정 위치

**파일**: `pages/admin/customers/index.tsx`

**수정 위치**: 
- 썸네일 그리드 부분 (3352-3450줄 근처)
- 각 썸네일 하단에 날짜 표시 및 수정 UI 추가

### 예상 작업 시간
- UI 추가: 1시간
- 상태 관리: 30분
- 테스트: 30분
- **총 예상 시간: 2시간**
