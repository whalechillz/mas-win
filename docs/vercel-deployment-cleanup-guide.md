# Vercel 배포 정리 가이드

## 목적
메인 브랜치 이외의 사용하지 않는 배포 버전을 Vercel 대시보드에서 삭제하는 방법

## Vercel 대시보드에서 배포 삭제하기

### 방법 1: 개별 배포 삭제 (가장 쉬운 방법) ✅

1. **Vercel 대시보드 접속**
   - https://vercel.com/taksoo-kims-projects/mas-win/deployments

2. **브랜치 필터 선택**
   - 상단의 "All Branches..." 드롭다운 클릭
   - 삭제하고 싶은 브랜치 선택 (예: `fix/tiptap-keyboard-input-and-...`)

3. **배포 삭제**
   - 각 배포 항목의 오른쪽 끝에 있는 `...` (점 3개) 메뉴 클릭
   - "Delete" 선택
   - 확인 대화상자에서 "Delete" 클릭

4. **반복**
   - 해당 브랜치의 모든 배포를 위 방법으로 삭제

5. **결과 확인**
   - 모든 배포를 삭제하면 브랜치 필터 목록에서 해당 브랜치가 사라집니다

### 방법 2: 일괄 삭제 (스크립트 사용)

```bash
# Vercel CLI 로그인
vercel login

# 특정 브랜치의 모든 배포 삭제
./scripts/remove-vercel-branch-deployments.sh <브랜치명>
```

## 삭제해도 안전한 배포

### ✅ 안전하게 삭제 가능
- **Preview 배포**: 모든 Preview 배포는 안전하게 삭제 가능
- **오래된 Production 배포**: 최신 Production 배포만 유지하고 나머지는 삭제 가능
- **삭제된 브랜치의 배포**: GitHub에서 이미 삭제한 브랜치의 배포는 모두 삭제 가능

### ⚠️ 주의해야 할 배포
- **현재 Production 배포**: 최신 Production 배포는 삭제하지 마세요
- **활발히 사용 중인 브랜치의 최신 배포**: 필요할 수 있으므로 확인 후 삭제

## 추천 정리 순서

1. **삭제된 브랜치의 배포 먼저 정리**
   - `fix/tiptap-keyboard-input-and-...` (이미 GitHub에서 삭제됨)
   - 다른 삭제된 브랜치들

2. **오래된 Preview 배포 정리**
   - 7일 이상 된 Preview 배포
   - 더 이상 필요 없는 테스트 배포

3. **오래된 Production 배포 정리**
   - 최신 Production 배포만 유지
   - 나머지는 삭제 (필요시 다시 배포 가능)

## 배포 삭제 후 확인

1. 브랜치 필터 목록에서 삭제한 브랜치가 사라졌는지 확인
2. 배포 목록이 깔끔하게 정리되었는지 확인
3. 필요한 배포만 남아있는지 확인

## 참고사항

- **배포 삭제는 되돌릴 수 없습니다** - 신중하게 삭제하세요
- **삭제된 배포의 URL은 더 이상 접근할 수 없습니다**
- **배포를 삭제해도 Git 히스토리는 유지됩니다** - 필요시 다시 배포 가능
- **Vercel은 자동으로 오래된 배포를 정리하는 기능도 제공합니다** (Deployment Retention 설정)
