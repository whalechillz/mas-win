## 📊 메시지 구성 설명

### 메시지 구조 (공통)
1. **개인화 인사**: `{name}` 변수 활용
2. **거리/위치 정보**: 거리 또는 위치 변수 활용
3. **특별 혜택**: 각 세그먼트별 맞춤 혜택
4. **강력한 CTA**: 링크와 전화번호 명확히 제시
5. **전문성 강조**: KGFA 1급 피팅 전문 상담

### 메시지 특징
- **거리 기반 개인화**: 각 고객의 거리에 맞는 메시지
- **위치 커스텀**: 원거리 고객에게는 `{province}` 변수로 지역별 맞춤
- **구매/비구매 구분 없음**: 모든 고객에게 동일한 메시지 적용
- **CTA 중심**: 명확한 행동 유도 (링크 + 전화번호)

### 변수 활용
- `{name}`: 고객명 (모든 메시지)
- `{distance_km}`: 거리 (50km 이내 메시지)
- `{province}`: 도 단위 (50km 이상 메시지)
  - 예: 제주, 울산, 경기, 충북, 경상, 전라 등

## 📝 변수 치환 가이드

### 변수 종류

1. **`{name}`**: 고객명
   - 자동 치환: 고객 DB의 `name` 필드
   - 예: "홍길동" → "홍길동님"

2. **`{distance_km}`**: 거리 (km)
   - 자동 치환: `customer_address_cache.distance_km`
   - 예: "25.3" → "약 25.3km 거리에"

3. **`{province}`**: 도 단위
   - 자동 치환: `customer_address_cache.province` 또는 주소에서 추출
   - 예: "제주" → "제주 고객님을 위한"
   - 예: "울산" → "울산 고객님을 위한"
   - 예: "경기" → "경기 고객님을 위한"

### 변수 치환 로직

**메시지 1 (50km 이내)**:
```javascript
message.replace(/{name}/g, customer.name)
       .replace(/{distance_km}/g, customer.distance_km)
```
- ✅ `{name}`: 고객명 자동 치환
- ✅ `{distance_km}`: 거리 정보 자동 치환 (소수점 첫째자리까지)

**메시지 2 (50km 이상)**:
```javascript
message.replace(/{name}/g, customer.name)
       .replace(/{province}/g, customer.province || extractProvince(customer.address))
```
- ✅ `{name}`: 고객명 자동 치환
- ✅ `{province}`: 지역 정보 자동 치환 (2026-01-21 수정 완료)
  - `customer_address_cache.province` 우선 사용
  - 없으면 주소에서 `extractProvince()` 함수로 추출
  - 예: "경기도 수원시" → "경기", "제주특별자치도" → "제주"

**메시지 3 (주소 없음)**:
```javascript
message.replace(/{name}/g, customer.name)
```
- ✅ `{name}`: 고객명 자동 치환
