# 회사 정보 배치 권장사항

## 📋 결론: 부가정보(선택사항)에 배치 권장 ✅

### 이유

1. **일관성**: 모든 템플릿에서 동일한 위치에 표시
2. **간결성**: 본문이 더 간결하고 핵심 내용에 집중
3. **관리 용이성**: 한 곳에서 관리, 수정 시 모든 템플릿에 자동 반영
4. **변수 불필요**: 고정 정보이므로 부가정보에 적합
5. **검수 안전성**: 본문이 짧아질수록 검수 통과 가능성 증가

---

## 비교 분석

### 옵션 1: 본문 하단에 배치

**장점:**
- 가독성 좋음 (메시지 버블 안에 크게 표시)
- 기존 템플릿과 일관성 유지

**단점:**
- 본문이 길어짐
- 모든 템플릿에 반복 입력 필요
- 수정 시 모든 템플릿 개별 수정 필요

### 옵션 2: 부가정보(선택사항)에 배치 ✅ **권장**

**장점:**
- 본문이 간결해짐
- 모든 템플릿에서 일관성 있게 표시
- 변수 불필요 (고정 정보이므로 적합)
- 템플릿 관리 용이 (한 곳에서 관리)
- 검수 통과 가능성 증가

**단점:**
- 메시지 하단에 작게 표시됨
- 가독성이 약간 떨어질 수 있음

---

## 적용 방법

1. **모든 템플릿에서 일관되게 적용**
   - 본문: 핵심 내용만 포함
   - 부가정보: 회사 정보 표준화

2. **스크립트 자동 입력**
   - `register-solapi-templates.js`에 부가정보 입력 로직 추가 완료
   - 자동으로 부가정보 필드에 회사 정보 입력

3. **기존 템플릿도 동일하게 적용**
   - 예약 접수 확인, 예약 확정 안내 등도 부가정보 사용 권장
