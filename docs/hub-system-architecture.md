# 허브 시스템 아키텍처

## 📊 허브 시스템 개요

MASGOLF의 허브 시스템은 **2가지 타입의 허브**로 구성됩니다:

### 1. 콘텐츠 중심 허브 (Content Hub)
**목적**: 주제 기반 콘텐츠를 여러 채널에 배포

**특성**:
- 주제 기반 (불규칙한 발행 주기)
- 하나의 허브 콘텐츠 → 여러 채널로 변형
- 채널: 블로그, SMS, 네이버 블로그, 카카오톡
- 저장: `cc_content_calendar` 테이블
- 관리: `/admin/content-calendar-hub` (콘텐츠 허브 탭)

**예시**:
- 주제: "PRO3 드라이버 고객 후기"
- 변형: 블로그 글, SMS, 네이버 블로그, 카카오톡 메시지

**현재 상태**: ✅ 구현 완료

---

### 2. 브랜드 홍보용 허브 (Branding Hub)
**목적**: 365일 일일 브랜드 노출 및 인지도 향상

**특성**:
- 365일 규칙적 발행
- 카카오톡 콘텐츠를 기본 허브로 사용
- 카카오톡 → 인스타그램, 유튜브 쇼츠, 틱톡으로 자동 변형
- 저장: `kakao_calendar` 테이블 (기본) + 확장 예정
- 관리: `/admin/content-calendar-hub` (데일리 브랜딩 탭)

**예시**:
- 카카오톡: 매일 프로필 + 피드 업데이트
- 인스타그램: 카카오톡 피드 이미지 → 인스타그램 피드
- 유튜브 쇼츠: 카카오톡 콘텐츠 → 쇼츠 영상
- 틱톡: 카카오톡 콘텐츠 → 틱톡 영상

**현재 상태**: 🚧 계획 단계

---

## 🎯 허브 시스템 구조

```
허브 시스템
├── 콘텐츠 중심 허브 (Content Hub)
│   ├── 허브 콘텐츠 생성
│   ├── 채널별 변형 생성
│   │   ├── 블로그 (홈피)
│   │   ├── SMS
│   │   ├── 네이버 블로그
│   │   └── 카카오톡
│   └── 멀티 채널 배포
│
└── 브랜드 홍보용 허브 (Branding Hub)
    ├── 카카오톡 콘텐츠 생성 (365일)
    ├── 자동 변형 시스템
    │   ├── 인스타그램 (예정)
    │   ├── 유튜브 쇼츠 (예정)
    │   └── 틱톡 (예정)
    └── 일일 브랜딩 배포
```

## 📋 브랜드 홍보용 허브 구현 계획

### Phase 1: 카카오톡 기본 허브 구축 ✅ (완료)
- 카카오톡 콘텐츠 365일 자동 생성 시스템
- 프로필 + 피드 콘텐츠 관리
- `/admin/kakao-content` 관리 페이지

### Phase 2: 인스타그램 자동 변형 (예정)
**목표**: 카카오톡 피드 이미지 → 인스타그램 피드 자동 변형

**구현 내용**:
- 카카오톡 피드 이미지 추출
- 인스타그램 최적화 (비율, 해상도 조정)
- 인스타그램 API 연동
- 자동 업로드 스케줄링

**기술 스택**:
- Instagram Graph API
- 이미지 변환 라이브러리
- 스케줄링 시스템

### Phase 3: 유튜브 쇼츠 자동 변형 (예정)
**목표**: 카카오톡 콘텐츠 → 유튜브 쇼츠 영상 자동 생성

**구현 내용**:
- 카카오톡 이미지 + 텍스트 → 영상 변환
- AI 음성 내레이션 추가
- 쇼츠 최적화 (9:16 비율, 60초 이내)
- 유튜브 API 연동
- 자동 업로드 스케줄링

**기술 스택**:
- YouTube Data API
- FFmpeg (영상 편집)
- TTS (Text-to-Speech)
- 영상 생성 라이브러리

### Phase 4: 틱톡 자동 변형 (예정)
**목표**: 카카오톡 콘텐츠 → 틱톡 영상 자동 생성

**구현 내용**:
- 카카오톡 이미지 + 텍스트 → 틱톡 영상 변환
- 틱톡 최적화 (9:16 비율, 3분 이내)
- 틱톡 API 연동
- 자동 업로드 스케줄링

**기술 스택**:
- TikTok API
- FFmpeg (영상 편집)
- 영상 생성 라이브러리

## 🔄 허브 간 관계

### 독립적 운영
- **콘텐츠 중심 허브**: 주제 기반, 불규칙 발행
- **브랜드 홍보용 허브**: 365일 규칙적 발행

### 공통점
- 모두 허브 시스템에서 관리
- 채널별 변형 지원
- 멀티 채널 배포

### 차이점
- **콘텐츠 중심**: 주제 → 여러 채널 변형
- **브랜드 홍보용**: 카카오톡 → 소셜 미디어 변형

## 📊 데이터 구조

### 콘텐츠 중심 허브
```sql
cc_content_calendar (
  id, title, summary, content_body, content_date,
  blog_post_id, sms_id, naver_blog_id, kakao_id,
  channel_status, hub_order, ...
)
```

### 브랜드 홍보용 허브 (현재)
```sql
kakao_calendar (
  id, date, account1_profile, account2_profile, feed, ...
)
```

### 브랜드 홍보용 허브 (확장 예정)
```sql
branding_hub (
  id, date, 
  kakao_content_id,
  instagram_post_id,
  youtube_shorts_id,
  tiktok_video_id,
  channel_status, ...
)
```

- [ ] 틱톡 자동 변형 시스템 구축

## 📝 참고 사항

- **"새 채널 추가" 모달 제거**: 동적 채널 추가 기능은 제거되었습니다.
- **채널 확장**: 향후 브랜드 홍보용 허브에서 인스타그램, 유튜브 쇼츠, 틱톡이 자동으로 추가됩니다.
- **허브 분리**: 콘텐츠 중심 허브와 브랜드 홍보용 허브는 독립적으로 운영되지만, 같은 허브 시스템에서 관리됩니다.
