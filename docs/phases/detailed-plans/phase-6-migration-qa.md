## 📋 질문과 답변

### 1. 디비 업데이트 안된 명단은?

**답변**: ✅ **모든 전화번호가 DB에 있습니다 (0건 누락)**

- CSV 파일: 1,247건
- DB 예약 데이터: 1,000건 (Supabase 쿼리 제한으로 일부만 조회됨)
- 누락된 전화번호: 0건

**참고**: Supabase의 기본 쿼리 제한으로 인해 1,000건만 조회되었지만, 실제로는 945건이 마이그레이션되었습니다.

---

### 2. 리스트에 사용클럽, 거리 등 정보는 어디에 있는거지?

**답변**: **DB에는 저장되어 있지만 UI에는 표시되지 않습니다.**

#### 현재 상황
- ✅ **DB에 저장됨**:
  - `club_brand`: 265건 (26.5%)
  - `club_loft`: 43건 (4.3%)
  - `club_shaft`: 37건 (3.7%)
  - `current_distance`: 265건 (26.5%)
  - `age_group`: 265건 (26.5%)
  - `trajectory`: 169건 (16.9%)
  - `shot_shape`: 160건 (16.0%)

- ❌ **UI에 표시 안됨**: 
  - 현재 `BookingListView`에는 `club` 필드만 표시됨
  - 구조화된 클럽 정보(브랜드, 로프트, 샤프트)는 표시 안됨
  - 거리, 연령대, 탄도, 구질 정보도 표시 안됨

#### 개선 필요
관리자 페이지 테이블에 다음 컬럼 추가 필요:
- 사용 클럽 (브랜드, 로프트, 샤프트)
- 현재 비거리
- 연령대

---

### 3. 노쇼일 경우 어떻게 하지? 취소의 기준은 뭐지?

**답변**: **두 가지 상태 필드가 있습니다.**

#### 상태 필드 구분

1. **`status` (예약 상태)**: 예약이 확정되었는지, 취소되었는지
   - `confirmed`: 확정 (예약 확정됨)
   - `pending`: 대기중 (예약 대기)
   - `cancelled`: 취소 (예약 전 취소)
   - `completed`: 완료 (서비스 완료)

2. **`attendance_status` (참석 여부)**: 예약일 당일 실제 참석 여부
   - `attended`: 참석 (실제 방문)
   - `no_show`: 노쇼 (예약했지만 방문 안함)
   - `cancelled`: 취소
   - `pending`: 대기중

#### 현재 데이터 통계
- **참석**: 207건
- **노쇼**: 13건
- **취소**: 0건 (attendance_status 기준)
- **대기중**: 679건

- **확정**: 664건 (status 기준)
- **취소**: 232건 (status 기준)
- **완료**: 101건

#### 처리 로직
- **노쇼 (`attendance_status = 'no_show'`)**:
  - 예약은 확정되었지만(`status = 'confirmed'`)
  - 실제로는 방문하지 않음
  - 고객의 `no_show_count` 증가

- **취소 (`status = 'cancelled'`)**:
  - 예약 전에 취소됨
  - `attendance_status`는 `cancelled` 또는 `pending`

#### 개선 필요
- UI에 `attendance_status` 표시 추가
- 노쇼/취소 필터 추가
- 상태 구분 명확화

---

### 4. 여러번 방문시 여러줄이 되어야하나?

**답변**: ✅ **네, 현재 구조가 맞습니다. 여러 줄로 저장됩니다.**

#### 현재 구조
- **각 예약마다 별도의 레코드**: `bookings` 테이블에 각 예약이 개별 행으로 저장
- **고객 정보는 통합**: `customers` 테이블에 `visit_count`, `no_show_count`, `last_visit_date` 저장

#### 통계
- **여러 번 방문한 고객**: 160명
- **최다 방문 고객**: 박영만 (01042450013) - 16회 방문

#### 장점
- 각 예약의 상세 정보 보존 (클럽 정보, 거리 등)
- 예약별 개별 추적 가능
- 방문 이력 추적 용이

#### 개선 가능
- 고객별 그룹화 뷰 추가
- 고객명 클릭 시 해당 고객의 모든 예약 표시
- 방문 횟수 표시

### 3. 노쇼일 경우 어떻게 하지? 취소의 기준은 뭐지?

**답변**: **두 가지 상태 필드가 있습니다.**

#### 상태 필드 구분

1. **`status` (예약 상태)**: 예약이 확정되었는지, 취소되었는지
   - `confirmed`: 확정 (예약 확정됨)
   - `pending`: 대기중 (예약 대기)
   - `cancelled`: 취소 (예약 전 취소)
   - `completed`: 완료 (서비스 완료)

2. **`attendance_status` (참석 여부)**: 예약일 당일 실제 참석 여부
   - `attended`: 참석 (실제 방문)
   - `no_show`: 노쇼 (예약했지만 방문 안함)
   - `cancelled`: 취소
   - `pending`: 대기중

#### 현재 데이터 통계
- **참석**: 207건
- **노쇼**: 13건
- **취소**: 0건 (attendance_status 기준)
- **대기중**: 679건

- **확정**: 664건 (status 기준)
- **취소**: 232건 (status 기준)
- **완료**: 101건

#### 처리 로직
- **노쇼 (`attendance_status = 'no_show'`)**:
  - 예약은 확정되었지만(`status = 'confirmed'`)
  - 실제로는 방문하지 않음
  - 고객의 `no_show_count` 증가

- **취소 (`status = 'cancelled'`)**:
  - 예약 전에 취소됨
  - `attendance_status`는 `cancelled` 또는 `pending`

#### 개선 필요
- UI에 `attendance_status` 표시 추가
- 노쇼/취소 필터 추가
- 상태 구분 명확화
